## What it does

Prometheus auto-routes every message — you never pick a mode, it decides:

| Trigger | Engine | Behaviour |
|---|---|---|
| Normal conversation | **big-pickle** (OpenCode Zen) | Streams the answer word-by-word |
| Message needs current info | **Tavily** → big-pickle | Searches the web, answers with cited `[n]` sources |
| You ask for a picture | **FLUX.2 klein** (Cloudflare Workers AI) | Generates an image from your prompt |
| You attach an image / PDF | **Gemini 3.1 Flash-Lite** (Google AI Studio) | Reads and analyzes the file |

The welcome-screen suggestion cards are generated by **Groq** (Llama).

It always speaks as **Prometheus** — cool, sassy, brutally honest — and will never disclose
the models or providers underneath.

## Features

- 🔐 **Real authentication** — email + password via **Supabase Auth** (managed). The session is
  carried in cookies and refreshed on every request by the middleware.
- 💬 **Persistent history** — every conversation is saved per-user in **Supabase Postgres**. Older
  chats can be **archived**, **restored**, or **deleted** from the collapsible sidebar. Each chat
  keeps its own context — old chats never bleed into new ones.
- ✨ **Random welcome** — a different big editorial greeting each visit; it vanishes the moment
  you send your first message.
- 📎 **Uploads** — drag PNG/JPEG/WEBP/GIF/PDF into the composer (≤ 20 MB). Bytes are stored in a
  private **Supabase Storage** bucket and served back through an authenticated route.
- ⌨️ **Word-by-word streaming** with a blinking gold cursor.
- 🎨 **Warm Editorial Luxury** — Cormorant Garamond + DM Sans, sharp edges, gold accents,
  floating shapes, reduced-motion aware.

## Running it

You need a **Supabase project** (the free tier is fine). Then:

```bash
npm install

# 1. Provision the database: open Supabase Dashboard → SQL Editor → New query,
#    paste the contents of supabase/schema.sql and Run it. This creates the
#    chats/messages/files tables, their RLS policies, and the private `uploads`
#    Storage bucket.

# 2. Fill in .env.local with your Supabase + provider keys (see Configuration).

npm run build
npm run start      # http://localhost:3000  (or `npm run dev` for hot reload)
```

Then open **http://localhost:3000**, create an account, and start talking. All data — users,
chats, messages and uploaded files — lives in your Supabase project.

## Configuration

All secrets live in `.env.local`. The Supabase URL and anon/publishable key are safe to expose
(they carry the `NEXT_PUBLIC_` prefix); everything else is server-only.

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL. |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public anon / publishable key — safe in the browser. |
| `SUPABASE_SERVICE_ROLE_KEY` | **Secret, server-only.** Used for all DB + Storage access; it bypasses RLS, so ownership is enforced in code (every query filters by `user_id`). |
| `OPENCODE_API_KEY` · `OPENCODE_BASE_URL` · `OPENCODE_MODEL` | Primary text model on OpenCode Zen (`big-pickle`). |
| `GOOGLE_API_KEY` · `GOOGLE_MODEL` | Vision/PDF model — `gemini-3.1-flash-lite` (used when files are attached). |
| `TAVILY_API_KEY` | Web search. |
| `CLOUDFLARE_API_TOKEN` · `CLOUDFLARE_ACCOUNT_ID` · `CLOUDFLARE_IMAGE_MODEL` | Image generation — `@cf/black-forest-labs/flux-2-klein-4b`. |
| `GROQ_API_KEY` · `GROQ_MODEL` | Dynamic welcome-screen suggestion cards. |

> `.env.local` is gitignored. Keep the `SUPABASE_SERVICE_ROLE_KEY` out of the browser and out of
> any copy you share — it grants full read/write to all user data and storage.

## Architecture

```
middleware.ts                         # refreshes the Supabase session on every request
app/
  api/
    auth/{register,login,logout,me}   # Supabase Auth (email + password)
    chats/ , chats/[id]/              # chat CRUD + archive/delete (Postgres)
    chat/                             # the streaming orchestrator (SSE)
    files/ , files/[id]/              # upload to Storage + authed serving
    suggestions/                      # Groq-generated welcome cards
  login/ register/ page.tsx           # auth screens
  page.tsx                            # the app (guarded)
lib/
  auth.ts                             # getCurrentUser() from the Supabase session
  supabase/
    server.ts                         # cookie-bound anon client (carries the user session)
    admin.ts                          # service-role client (server-only) + `uploads` bucket
  ai/
    prompt.ts                         # Prometheus identity system prompt
    providers.ts                      # text/vision streams, Tavily, FLUX, Groq, intent router
components/                           # ChatApp, Sidebar, Message, Composer, AuthForm, …
supabase/
  schema.sql                          # tables, indexes, RLS policies, Storage bucket
```

The orchestrator (`app/api/chat/route.ts`) persists the user message, classifies intent with
local heuristics, runs the chosen engine, streams Server-Sent Events to the client, then persists
the assistant turn — so a dropped connection never loses a completed message. All data access goes
through the service-role client and re-checks ownership in code; Row Level Security in
`supabase/schema.sql` is enabled as defense-in-depth.
